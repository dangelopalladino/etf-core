# .impeccable.md

Project-specific instructions for `/impeccable` workflows on `@dangelopalladino/etf-core`.

This file is loaded by every `/impeccable` command before any design action. It describes the audience, register, and constraints that the codebase alone cannot reveal. The structural design system (tokens, components, elevation, motion) lives in `DESIGN.md` and `DESIGN.json` at the project root — read those for hard values. This file holds the *why*.

---

## Project Identity

`etf-core` is a published npm package on GitHub Packages (`@dangelopalladino/etf-core`, currently v1.11.x). It is the canonical design system + shared business logic layer for two sibling Next.js + AntD sites:

- **6identities.com** (consumer) — the 6 Identities® diagnostic for former athletes
- **etfframework.com** (professional) — the Executable Transition Framework™ for therapists, coaches, and practitioners working with athletes after the career ends

Both sites stay on independent Vercel projects and independent repos. This package is the single dependency that prevents drift between them. Every change here ships to both sites at the next version bump.

## Design Context

### Users

**End users of the consuming sites — two distinct audiences served by one token plate:**

- **6identities.com users** — former athletes (NCAA, pro, Olympic, junior, combat, club). Post-career. Reading on phones at 10 a.m. on a Tuesday because the training schedule no longer exists. They have functioning pattern recognition for what a wellness app looks like and what an athlete brand looks like; they are scanning for signals that this is the second category. They expect Players' Tribune editorial weight, Whoop-grade information density, Bloomberg Terminal precision in numbers and scores.

- **etfframework.com users** — therapists, coaches, and practitioners (Tier 1 ETF Certified Practitioner™ candidates) plus institutional sports orgs (future Tier 2 licensees). They evaluate the site the way they evaluate Mayo Clinic, McKinsey Insights, or a peer-reviewed methodology paper. They are scanning for clinical seriousness and scope-of-practice clarity. They will not enroll if the site reads like a coaching funnel.

**Direct users of this package — engineers building both consumer apps**, plus future Claude Code sessions extending the design system. The package's API surface (token exports, AntD theme configs, server/client primitives) is itself a design surface. Cleanliness, predictability, and explicit naming matter as much as the visuals they produce.

**Job to be done.** Publish two distinct visual registers from one source of truth without either site feeling like a generic version of the other, and without engineers having to think about which token bundle to import — the namespacing should make it obvious.

### Brand Personality

**Voice: Disciplined. Editorial. Anti-decorative.**

- **Disciplined** — every token has one job. Every component has bounded variants. The system says no often. Tokens are added through review, not by accretion.
- **Editorial** — typographic hierarchy carries meaning. Spacing has rhythm. The reader's eye is led, not nudged. Asymmetry where it earns it, never for novelty.
- **Anti-decorative** — gradients, glows, blur, shadow on content surfaces, sparklines as decoration, illustrations of brains-with-hearts: all banned. Decoration is the failure mode this brand is positioned against.

**Emotional goals.** The reader (athlete or practitioner) lands on a page, reads three sentences, and thinks *this is the first time anyone has described what I'm actually dealing with.* The visual layer must not undermine that moment. Confidence without warmth-as-decoration. Authority without coldness. Restraint that reads as respect for the reader's intelligence.

**Anti-personality.** Not pleasant. Not warm. Not delightful. Not playful. Not bouncy. Not friendly-feeling. Not motivational. The brand is for adults navigating a real systems failure; the design respects that by refusing the patterns that would soften it.

### Aesthetic Direction

**North Star:** *The Athletic Periodical* — editorial spine that holds two voices on one press. Players' Tribune long-form crossed with the Bloomberg Terminal manual. Disciplined typographic hierarchy, confident asymmetry, zero decoration.

**Reference brands (for tone, not literal mimicry):**
- 6id: Players' Tribune, Whoop Journal, The Athletic, Andre Agassi's *Open*
- etfframework: Bloomberg Terminal, Mayo Clinic, McKinsey Insights, NEJM, peer-reviewed methodology papers

**Anti-references (instant rewrite triggers):**
- Therapist-office aesthetics (cream backgrounds, muted teal, dusty rose, sage, blush)
- Wellness-app patterns (pill buttons, rounded everything, soft shadows, gradients, breathwork imagery)
- Coaching-funnel visual language (smiling-people-on-couch photography, three-adjective stacks, motivational color)
- Generic SaaS templates (three-column equal-width card grids with icon-headline-text, hover-shadow on cards)
- AI design tells (cyan-on-dark, purple-to-blue gradients, glassmorphism, sparkline decoration, gradient text, left-stripe accent borders, neon-on-dark)
- Manosphere aesthetics (alpha/sigma/grind imagery, dark+orange hyper-masculine palettes)

**Palette discipline.**
- 6id: navy + soft off-white + Electric Cobalt (`#1877F2`). Identity-color palette (Deep Cobalt, Burnt Amber, Forest Teal, Slate Violet, Warm Gold, Vivid Emerald) used **only** inside the type system — type cards, badges, result reveals.
- etfframework: Deep Forest + warm white + Aged Brass CTA. Ecosystem Navy at footer for cross-link to 6id.
- Both sides: pure `#000` and `#FFF` are banned. Neutrals tint toward the bundle's brand hue.
- The headline-accent rule: exactly one word per headline takes the accent color. Two is dilution; zero is generic.

**Theme.** Light-first on both sides today. 6id has a defined dark palette reserved for future opt-in. The etfframework practitioner portal is the only future surface authorized for dark variant.

**Typography.** General Sans (Indian Type Foundry, commercial license on file) is the body and UI face on both sides. Source Serif 4 sets headings on etfframework. JetBrains Mono carries data values, eyebrows on 6id. Weights 400 and 600 only (500 permitted on 6id mono labels and stat values). Weights ≥ 700 do not ship.

**Elevation.** Hairline borders default. Navy-tinted shadow at the overlay tier only (Modal, Popover, Drawer). Cards stay flat. Hover never elevates with shadow — it changes color or border.

**Motion.** Restrained. 120–180ms hover. 200–280ms state changes. 600ms reveals reserved for earned moments (assessment results stagger, post-purchase confirmation). No scroll-linked animation. No spinners. Animate `transform` and `opacity` only. Respect `prefers-reduced-motion`.

### Design Principles

1. **Discipline beats novelty.** Two production sites consume this package. New tokens require sister-token review across both bundles. New components require both 6id and etfframework theme verification. Visual additions that work in only one bundle do not ship.

2. **The asymmetry is the system.** 6id and etfframework are deliberately not symmetrical. 6id is editorial, athlete-brand, navy-anchored, Electric Cobalt action. etfframework is institutional, white-paper, forest-anchored, Aged Brass CTA. Treat the two bundles as two publications from one press, not two themes of one product. If a change makes them feel more alike, push back.

3. **Content surfaces are flat. Overlays earn shadow.** Hairline border + tone tint is the default container expression. Shadow appears only on Modal, Popover, Drawer, Dropdown, Tooltip — never on Cards, Heroes, NoticeCards, or content surfaces. Hover never adds shadow.

4. **Exactly one accent word per headline.** This is the load-bearing typographic move on both sides. Apply at the component level (Hero, SectionHeader, ServerHeading consumers). Never two; never zero.

5. **The brand bible vetoes drift.** When implementation conflicts with `etf-brand-design` / `etf-brand-messaging` / `etf-brand-shared` skill guidance, the skill wins for register and direction; the implemented tokens win for code consumption today. Surface the divergence and propose a token migration rather than introducing ad-hoc values. Three known conflicts are documented in `DESIGN.md → Colors → Brand-bible reconciliation`.

## Workflow Notes

- **Read DESIGN.md first** on every `/impeccable` invocation. It holds the hard values (color hex, type scale clamps, spacing scale, radius per bundle, AntD overrides). DESIGN.json is the machine-readable sidecar with the same data.
- **Read this file second** for the audience and register that DESIGN.md does not encode.
- **For copy** (any words shipped under the brand), invoke the `etf-brand-messaging` skill for the relevant site's voice file — do not write voice from `.impeccable.md` alone.
- **For visual review** (audit, polish, critique), invoke the `etf-brand-design` skill alongside `/impeccable audit` so the audit checklist includes the visual kill list from the brand bible.
- **Token changes:** every PR that modifies `src/tokens/*` requires verification against both bundles. CI ships ESLint enforcement of off-grid violations (the v1.11 ratchet) and motion keyframe allowlist. Do not bypass either.
- **Brand-bible-vs-tokens conflicts:** if a `/impeccable` action would introduce a value that diverges from the brand bible (Electric Cobalt vs `#2D5BFF`, button pill vs 4px, shadow on Cards), pause and surface the conflict. Prefer the bundle override path over inventing new tokens.
